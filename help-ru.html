Справочное руководство
Рабочий журнал
Инструмент для ежедневной фиксации выполненных работ. Хранит записи, фотографии, категории и историю изменений — полностью локально, без подключения к интернету.
● v1.1  ·  Работает полностью офлайн
🚀
Быстрый старт
#start
1
Запустите сервер
Откройте терминал в папке проекта и выполните node server.js. Требуется Node.js 16+. Интернет не нужен.
2
Откройте браузер
Перейдите по адресу http://localhost:3000. Используйте Chrome, Firefox, Edge или любой современный браузер.
3
Создайте категории
Перейдите в раздел Категории и добавьте нужные группы работ (например: Электрика, Сантехника, Кровля). Это необязательно, но удобно.
4
Добавьте первую запись
Нажмите + Новая запись в разделе Журнал, заполните поля и сохраните. База данных хранится прямо в браузере.
🔒
Полная офлайн-работа. Приложение не делает ни одного запроса в интернет. Все данные (записи, категории, фотографии, шрифты) хранятся и обрабатываются исключительно на вашем компьютере.
📋
Журнал
#journal

Главная страница приложения. Показывает все записи в обратном хронологическом порядке — от новых к старым.

🔍
Поиск
Полнотекстовый поиск по описанию работы, выполненным действиям и названию категории. Совпадения подсвечиваются прямо в карточках.
🏷
Фильтр по категории
Выберите категорию из выпадающего списка — журнал мгновенно покажет только записи этой категории.
📅
Фильтр по периоду
Задайте начальную и конечную дату, чтобы увидеть записи за конкретный диапазон. Фильтры можно комбинировать.
✕
Сброс фильтров
Кнопка «Сброс» мгновенно очищает все активные фильтры и возвращает полный список записей.

Каждая карточка в журнале показывает дату, категорию, описание работы, краткий список выполненных действий и миниатюры фотографий (если есть). Кнопки управления карточкой появляются при наведении курсора.

✏️
Форма записи
#entry-form

Форма создания и редактирования записи открывается в модальном окне. Все поля доступны сразу без переключения вкладок.

Поле Описание
Дата* Дата выполнения работы. Заполняется автоматически текущей датой при создании новой записи.
Категория Необязательное поле. Выберите из списка категорий, созданных заранее. Помогает фильтровать и группировать записи в отчётах.
Описание работы* Краткое название задачи или объекта — то, что будет видно в заголовке карточки. Например: «Замена счётчика», «Ремонт кровли секция 3».
Выполненные действия Подробное описание того, что именно было сделано. Поддерживает многострочный ввод. Отображается в карточке и в отчётах.
Запчасти / Материалы Динамический список позиций. Каждая строка содержит наименование и количество. Кнопка «+ добавить позицию» добавляет новую строку. Строку можно удалить кнопкой ✕ справа.
Фотографии До 10 фотографий на запись. Форматы: JPEG, PNG, GIF, WebP. Максимальный размер файла: 10 МБ. Подробнее — в разделе «Фотографии».
💡
Поля помеченные * — обязательные. Сохранить запись без них не получится — появится подсказка.
⌨
Нажмите Ctrl + Enter для быстрого сохранения, Esc — для отмены и закрытия формы.
📷
Фотографии
#photos

К каждой записи можно прикрепить до 10 фотографий. Файлы хранятся в папке photos/ внутри проекта на диске — не в базе данных браузера.

1
Откройте форму записи
Создайте новую запись или нажмите ✎ на существующей.
2
Нажмите на кнопку добавления фото
В нижней части формы — секция «Фотографии» с кнопкой с пунктирной рамкой. Можно выбрать несколько файлов сразу.
3
Дождитесь загрузки
Фото загружаются на локальный сервер и сразу появляются в виде миниатюр. Можно добавлять по одному или сразу несколько.
4
Сохраните запись
Фотографии привязываются к записи при сохранении. Если нажать «Отмена» — только что загруженные фото автоматически удаляются.
🗑
Удаление фото
Наведите курсор на миниатюру — появится кнопка ✕. Удалённое фото помечается, файл стирается с диска при сохранении записи.
🔍
Полноэкранный просмотр
Кликните по любой миниатюре — откроется лайтбокс. Листайте стрелками или клавишами ← →. Закрытие — Esc или клик на фон.
⚠
При переносе на другой компьютер обязательно копируйте папку photos/ вместе с остальными файлами проекта. Без неё фотографии в записях не отобразятся.
👁
Просмотр записи
#view

Кнопка «👁 Подробнее» в карточке журнала открывает полноэкранную модалку с полным содержимым записи.

  • Дата, категория и заголовок записи
  • Полный текст выполненных действий
  • Список запчастей и материалов с количеством
  • Сетка фотографий с возможностью открыть лайтбокс
  • Кнопка «Редактировать» — сразу переходит в режим правки
💡
Кнопки управления карточкой (Подробнее, редактирование, удаление) появляются при наведении курсора на карточку.
🏷
Категории
#categories

Категории помогают группировать записи по типу работ. Рекомендуется создать категории до начала ведения журнала.

1
Перейдите в раздел «Категории»
Нажмите в боковом меню слева.
2
Нажмите «+ Добавить»
Введите название категории. Примеры: Электрика, Сантехника, Кровля, ОВиК, Слаботочка.
3
Используйте при создании записей
Категория появится в выпадающем списке в форме новой записи и в фильтрах журнала.
💡
На каждой карточке категории отображается счётчик записей. При удалении категории — сами записи не удаляются, только поле «категория» у них очищается.
📊
Отчёты
#reports

Раздел формирования печатных отчётов о выполненных работах за выбранный период.

⚡
Быстрые периоды
Четыре кнопки для мгновенного выбора периода: эта неделя, этот месяц, этот квартал, этот год.
🗓
Произвольный период
Задайте любые начальную и конечную даты вручную. Предварительный просмотр обновляется немедленно.
🏷
Фильтр категорий
Сформируйте отчёт только по одной категории или по всем сразу.
🖨
Печать / PDF
Кнопка «Печать» открывает готовый документ. В диалоге печати браузера выберите «Сохранить как PDF» вместо принтера.

В отчёте записи группируются по датам. Для каждой записи указываются: описание, категория, выполненные действия и список использованных материалов. Фотографии в отчёт не включаются.

💡
При открытии раздела «Отчёты» автоматически выбирается текущий месяц и сразу показывается предварительный просмотр.
🗄
Архив снимков
#archive

Система автоматического резервного копирования базы данных. Снимки создаются без участия пользователя в критических ситуациях, а также вручную по запросу.

Каждый снимок содержит полную копию всех записей и категорий на момент создания.

Тип снимкаКогда создаётся
Перед очисткой Автоматически при нажатии «Очистить все данные» в настройках — до удаления.
Перед импортом Автоматически при загрузке JSON-файла — до замены существующих данных.
Перед восстановлением Автоматически при восстановлении из другого снимка — текущее состояние сохраняется.
Ручной снимок По кнопке «📷 Создать снимок» на странице архива — в любой момент по вашему желанию.
↺
Восстановление
Кнопка «↺ Восстановить» заменяет текущую базу данными из снимка. Текущее состояние перед этим автоматически сохраняется.
↓
Скачать снимок
Любой снимок можно скачать как JSON-файл и перенести на другой компьютер через Импорт.
ℹ
Хранится не более 30 снимков. При превышении лимита самый старый снимок удаляется автоматически. Снимки не включают фотографии — только текстовые данные записей.
💾
Перенос данных
#transfer

Данные хранятся в двух местах: текстовые данные — в IndexedDB браузера, фотографии — в папке photos/ на диске. Для полного переноса нужно перенести и то, и другое.

1
Экспортируйте базу данных
Нажмите «↑ Экспорт» в верхней панели или в Настройках. Скачается файл work-journal-ДАТА.json.
2
Скопируйте папку photos/
Скопируйте всю папку photos/ из проекта. В ней хранятся все загруженные фотографии.
3
На новом компьютере запустите сервер
Скопируйте весь проект, поместите папку photos/ на место, запустите node server.js.
4
Импортируйте JSON
Нажмите «↓ Импорт» и выберите сохранённый JSON-файл. Все записи и категории восстановятся.
💡
Самый простой способ переноса — скопировать всю папку проекта целиком. IndexedDB и папку photos/ скопируйте с данными, иначе часть информации будет потеряна.
⚙
Настройки
#settings
ПараметрОписание
Язык интерфейса Переключение между русским, английским и польским. Влияет только на интерфейс — данные (записи, категории) языком не затрагиваются. Выбор сохраняется между сессиями.
Экспорт базы Скачать все записи и категории в JSON-файл. Используется для резервного копирования и переноса на другой компьютер.
Импорт базы Загрузить данные из JSON-файла. Внимание: текущие данные будут заменены. Перед импортом автоматически создаётся снимок архива.
Очистить все данные Удаляет все записи и категории из базы данных браузера. Внимание: перед очисткой автоматически создаётся снимок архива. Фотографии в папке photos/ не удаляются.
