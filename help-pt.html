Manual do usuário
Work Journal
Registro do trabalho realizado: entradas, fotos, categorias e histórico — tudo local, sem internet.
● v1.1  ·  Totalmente offline
🚀
Início rápido
#start
1
Inicie o servidor
Na pasta do projeto execute node server.js. Node.js 16+. Sem internet.
2
Abra o navegador
Acesse http://localhost:3000. Chrome, Firefox, Edge ou outro navegador moderno.
3
Crie categorias
Em Categorias adicione grupos de trabalho. Opcional, mas útil.
4
Primeiro registro
Em Diário, + Novo registro, preencha e salve. Banco no navegador (IndexedDB).
🔒
Totalmente offline. Nenhum dado é enviado à internet. Tudo fica no seu computador.

Diário
#journal

The main page of the application. Shows all entries in reverse chronological order — newest first.

🔍
Search
Full-text search across work description, performed actions and category name. Matches are highlighted directly in the cards.
🏷
Category Filter
Select a category from the dropdown — the journal instantly shows only entries from that category.
📅
Date Range Filter
Set a start and end date to see entries for a specific period. Filters can be combined freely.
✕
Reset Filters
The «Reset» button instantly clears all active filters and restores the full entry list.

Each journal card shows the date, category, work description, a summary of actions performed, and photo thumbnails (if any). Card action buttons appear on hover.

✏️
Formulário de registro
#entry-form

The create/edit form opens in a modal window. All fields are available at once without switching tabs.

FieldDescription
Date*Date the work was performed. Automatically filled with today's date when creating a new entry.
CategoryOptional. Select from the list of pre-created categories. Helps filter and group entries in reports.
Work Description*A short name for the task or object — shown as the card heading. E.g.: "Meter replacement", "Roof repair section 3".
Actions PerformedDetailed description of what was done. Supports multi-line input. Shown in the card and in reports.
Parts / MaterialsDynamic list of items. Each row has a name and quantity. The «+ add item» button adds a new row. Remove a row with the ✕ button.
PhotosUp to 10 photos per entry. Formats: JPEG, PNG, GIF, WebP. Max file size: 10 MB. See the Photos section for details.
💡
Fields marked * are required. You cannot save without them — a hint will appear.
⌨
Press Ctrl + Enter to save quickly, Esc to cancel and close the form.
📷
Fotos
#photos

Up to 10 photos can be attached to each entry. Files are stored in the photos/ folder inside the project on disk — not in the browser database.

1
Open the entry form
Create a new entry or click ✎ on an existing one.
2
Click the photo add button
At the bottom of the form — the «Photos» section with a dashed-border button. You can select multiple files at once.
3
Wait for upload
Photos upload to the local server and appear as thumbnails immediately. You can add them one by one or several at once.
4
Save the entry
Photos are linked to the entry when saved. If you click «Cancel», any newly uploaded photos are automatically deleted.
🗑
Removing photos
Hover over a thumbnail — a ✕ button appears. The deleted photo is marked; the file is removed from disk when the entry is saved.
🔍
Full-screen view
Click any thumbnail to open a lightbox. Navigate with arrow buttons or ← → keys. Close with Esc or click the background.
⚠
When moving to another computer, always copy the photos/ folder along with the rest of the project. Without it, photos in entries will not display.
👁
Detalhes do registro
#view

The «👁 Details» button on a journal card opens a full-screen modal with the complete entry content.

  • Date, category and entry heading
  • Full text of actions performed
  • Parts and materials list with quantities
  • Photo grid with lightbox support
  • «Edit» button — jumps directly to edit mode
💡
Card action buttons (Details, edit, delete) appear when you hover over the card.
🏷
Categorias
#categories

Categories help group entries by type of work. It is recommended to create categories before starting to use the journal.

1
Go to «Categories»
Click in the left sidebar.
2
Click «+ Add»
Enter a category name. Examples: Electrical, Plumbing, Roofing, HVAC, Low-voltage.
3
Use when creating entries
The category will appear in the dropdown in the new entry form and in journal filters.
💡
Each category card shows an entry count. When a category is deleted — entries are not deleted, only their «category» field is cleared.
📊
Relatórios
#reports

The report section lets you generate printable documents about completed work for a selected period.

⚡
Quick Periods
Four buttons for instant period selection: this week, this month, this quarter, this year.
🗓
Custom Period
Set any start and end dates manually. The preview updates instantly.
🏷
Category Filter
Generate a report for a single category or all at once.
🖨
Print / PDF
The «Print» button opens the ready document. In the browser print dialog, select «Save as PDF» instead of a printer.

In the report, entries are grouped by date. For each entry: description, category, actions performed and materials list are shown. Photos are not included in reports.

💡
When opening the Reports section, the current month is automatically selected and a preview is shown immediately.
🗄
Arquivo de instantâneos
#archive

An automatic backup system for the database. Snapshots are created automatically in critical situations and manually on demand. Each snapshot contains a full copy of all entries and categories.

Snapshot typeWhen created
Before clearAutomatically when clicking «Clear all data» in settings — before deletion.
Before importAutomatically when loading a JSON file — before existing data is replaced.
Before restoreAutomatically when restoring from another snapshot — current state is saved first.
Manual snapshotVia the «📷 Create snapshot» button on the archive page — at any time.
↺
Restore
The «↺ Restore» button replaces the current database with data from the snapshot. The current state is automatically saved first.
↓
Download snapshot
Any snapshot can be downloaded as a JSON file and transferred to another computer via Import.
ℹ
Up to 30 snapshots are stored. When the limit is exceeded, the oldest snapshot is deleted automatically. Snapshots do not include photos — only text entry data.
💾
Transferência de dados
#transfer

Data is stored in two places: text data — in the browser's IndexedDB, photos — in the photos/ folder on disk. A full transfer requires moving both.

1
Export the database
Click «↑ Export» in the top bar or in Settings. A file work-journal-DATE.json will download.
2
Copy the photos/ folder
Copy the entire photos/ folder from the project. It contains all uploaded photos.
3
Start the server on the new computer
Copy the whole project, place the photos/ folder in position, run node server.js.
4
Import the JSON
Click «↓ Import» and select the saved JSON file. All entries and categories will be restored.
💡
Easiest transfer method — copy the entire project folder. Copy the IndexedDB and photos/ folder with data, otherwise some information will be lost.
